# HDMS — ระบบจัดการภัยพิบัติทางหลวง (Highways Disaster Management System)

## Overview
**HDMS** (Highways Disaster Management System) คือระบบจัดการภัยพิบัติทางหลวง พัฒนาโดยศูนย์บริหารงานอุบัติเหตุ (Disaster Emergency Management) ของ[[กรมทางหลวง]] เป็นผลงานนวัตกรรมด้าน Service Delivery ภายใต้แนวคิด Digital Highway

## Pain Point
- ภัยพิบัติทางธรรมชาติ (น้ำท่วม ดินถล่ม) ส่งผลกระทบต่อโครงข่ายทางหลวงทั่วประเทศ
- ขาดระบบรวมศูนย์ในการติดตามและรายงานสถานการณ์แบบ Real-time
- การประสานงานระหว่างหน่วยงานเมื่อเกิดเหตุยังล่าช้า

## วิธีการทำงาน
1. **หน่วยตรวจ/เฝ้าระวัง** — ตรวจสอบพื้นที่จุดเสี่ยงเกิดเหตุภัยพิบัติ
2. **รายงานเหตุการณ์** — ผ่าน Line OA / Website
3. **กรณีเร่งด่วน** — รายงานอัตโนมัติผ่าน Website
4. **ศูนย์อุบัติเหตุ** — รวบรวม Dashboard ติดตามสถานการณ์แบบ Real-time
5. **แจ้งเตือน** — เหตุการณ์สำคัญผ่าน Line OA ทันที

### ฟีเจอร์หลัก
| ฟีเจอร์ | รายละเอียด |
|---|---|
| แผนที่ภัยพิบัติ (Disaster Map) | แสดงจุดเสี่ยงทั่วประเทศ |
| Dashboard | สรุปสถานการณ์แบบ Real-time |
| Warning System | แจ้งเตือนอัตโนมัติ |
| Drone Survey | ภาพถ่ายทางอากาศจุดเกิดเหตุ |
| แผนที่ฝนตก/น้ำท่วม | ข้อมูลปริมาณฝนและจุดอันตราย |

## ผลการดำเนินงาน
- จำนวนเส้นทางเฝ้าระวัง: **141 เส้นทาง** (สูงสุด), **59 เส้นทาง** (สูง), **4 เส้นทาง** (ปานกลาง)
- จังหวัดที่มีความเสี่ยงสูง: สระบุรี, สุโขทัย, นครศรีธรรมราช
- งบประมาณฟื้นฟูผิวทางจากภัยพิบัติ: ~**20,000 ล้านบาท**
- ใช้ประสานงานกับหน่วยงานที่เกี่ยวข้อง 6+ หน่วยงาน
