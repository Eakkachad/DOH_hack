# Source: CarbonWay Development Guide

The team handbook for the CarbonWay hackathon sprint. A practical operational manual covering the folder structure, what each person does to start, how to run the prototype, daily sync schedule, key numbers reference, data flow diagram between team members, and troubleshooting.

**Key contents:**
- Quick-start instructions per person
- Full folder map with ownership
- Prototype startup commands (npx serve, Python http.server)
- Daily sync schedule (4 days, 18:00 each day)
- Data flow diagram (P1 → P2, P1 → P3)
- Frozen constants (field names, intervention IDs, threshold values)
- Troubleshooting table
